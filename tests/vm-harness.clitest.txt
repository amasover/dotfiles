pytest's job; these cover the shell seams: the scrub log filter (via the
`vm-harness scrub` subcommand), the display tool's pass-through guarantees,

clitest format notes: `$ ` lines are commands, following lines are expected
output, a lone `$` ends a block; prose like this is ignored outside blocks.

--- scrub: caret-notation terminal replies (guest tty echoctl) stripped ---

The guest tty echoes terminal replies — cursor-position (...R) and
window-size (...t) reports answering systemd's queries — as literal '^[['
bytes with NO ESC, so the \x1b patterns never match them (seen live
2026-07-10 in the boot serial stream).

$ printf '%s\n' '^[[56;192R^[[56;192R[  OK  ] Finished Initial Setup.' '^[[8;25;94t^[[25;94Rplain tail' | .local/bin/setup/vm-harness scrub
[  OK  ] Finished Initial Setup.
plain tail
$

--- display tool: no controlling tty -> silent degradation, byte-faithful ---

setsid drops the controlling terminal, so /dev/tty can't open and the tool
must become a pure pass-through (exit 0, output identical to input).

$ t=$(mktemp -d) && printf 'raw \033[31mred\033[0m\rframe A\rframe B\n==> Stage one\nHARNESS-CLOUDINIT-UP\n' > "$t/in" && setsid .local/bin/tools/vm-harness-display --mode bar --phase test < "$t/in" > "$t/out" 2> "$t/err"; echo "rc=$?"; cmp -s "$t/in" "$t/out" && echo byte-faithful; cat "$t/err"
rc=0
byte-faithful
$

--- display tool: bar mode under a real pty still passes bytes through ---

script(1) provides the pty; the stdout leg (what scrub/the state log would
see) must stay byte-identical, and the pty session must contain the phase
breadcrumb (done tick, CURRENT uppercase, pending plain), the opinionated
bootstrap stage mapped from the say-line, and the parsed detail anchor.

$ t=$(mktemp -d) && printf '==> Package reconcile (metapac shows its plan and prompts)\n:: Retrieving packages...\ndata line\n' > "$t/in" && script -qec "sh -c '.local/bin/tools/vm-harness-display --mode bar --phase bootstrap --phases create:done,install:done,boot:done,bootstrap:current,check:pending < $t/in > $t/out'" "$t/session" > /dev/null 2>&1; echo "rc=$?"; cmp -s "$t/in" "$t/out" && echo stdout-leg-byte-faithful
rc=0
stdout-leg-byte-faithful
$

--- display tool: compact mode passes bytes through, suppresses the firehose ---

The pty session should hold only status-line redraws — far smaller than a
firehose mirror would be; assert the raw data line never reaches the tty.

$ t=$(mktemp -d) && printf 'unique-firehose-marker-1\n==> Stage two\n' > "$t/in" && script -qec "sh -c '.local/bin/tools/vm-harness-display --mode compact --phase check < $t/in > $t/out'" "$t/session" > /dev/null 2>&1; cmp -s "$t/in" "$t/out" && echo byte-faithful; grep -c 'unique-firehose-marker-1' "$t/session"
